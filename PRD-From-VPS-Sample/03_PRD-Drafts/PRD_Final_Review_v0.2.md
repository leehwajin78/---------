### 4. 비기능 — ✅ PASS

**패스 조건:** 성능·보안·가용성·비용에 임계치 + 모니터링 항목 명시

| NFR 카테고리 | 임계치 수 | 모니터링 항목 | 판정 |
| :--- | :---: | :---: | :---: |
| **성능** (§5-1) | 4개 (p95 ≤ 15초, ≤ 10분, LCP ≤ 2.5초, ≥ 50 req/hr) | Vercel Functions Log (p95 > 20초 알림) | ✅ |
| **신뢰성** (§5-2) | 4개 (≥ 99.0%, ≤ 1.0%, 일 1회 백업, 4시간 SLA) | Vercel Analytics (5xx ≥ 3건 알림) | ✅ |
| **보안** (§5-3) | 5개 + 테스트 기준 컬럼 (SSL A+, 하드코딩 0건, OWASP 0건) | IP 차단 로그 즉시 알림 | ✅ |
| **비용** (§5-4) | 4개 + 상한 초과 대응 ($100 자동 차단, DB 80% 아카이빙) | Anthropic Dashboard ($5/일, $100/월) | ✅ |
| **모니터링** (§5-5) | — | 7개 항목 (5xx, 응답시간, DB 실패, API 비용, 폼 완료율, AI 환각율, 불성실 비율) | ✅ |

**임계치 ↔ 알림 연결 정합성 체크:**

| 임계치 | 알림 연결 | 매칭 |
| :--- | :--- | :---: |
| p95 ≤ 15초 (진단 리포트) | p95 > 20초 Slack 알림 | ⚠️ 미세 불일치 — 알림 기준(20초)이 SLO(15초)보다 넓음 |
| 5xx ≤ 1.0% | 5분간 5xx ≥ 3건 Slack 알림 | ✅ 합리적 |
| Claude API ≤ $50/월 | $5/일 이메일, $100/월 자동 차단 | ✅ 이중 장치 |
| AI 환각 < 5% (AC1 SLO) | 월간 환각율 > 10% 프롬프트 튜닝 | ⚠️ 알림 임계치(10%)가 SLO(5%)보다 넓음 |

> **참고:** 경미한 불일치 2건 발견 (응답 시간 알림 20초 vs SLO 15초, 환각율 알림 10% vs SLO 5%). 실운영에서는 SLO 근접 시 "경고", 초과 시 "긴급"으로 2단계 알림 분리를 권장하나, MVP 단계에서는 현재 단일 임계치로도 운영 가능. Phase 0 착수에 무방.

**소결:** 5개 NFR 카테고리 전체에 수치 임계치·모니터링 완비. 경미한 알림 임계치 불일치 2건은 V2에서 보완. **PASS**.

| 설계 결정 | PRD 내 기재 위치 | 결정 이유 | 대안 언급 |
| :--- | :--- | :--- | :---: |
