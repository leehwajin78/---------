## 🧪 Acceptance Criteria (BDD/GWT)

**Scenario 1: 로컬 SQLite 환경에서 Prisma 초기화 성공**
- Given: 프로젝트에 `prisma/schema.prisma`가 생성되고 `DATABASE_URL`이 SQLite로 설정된 상태
- When: `npx prisma generate`를 실행하면
- Then: `node_modules/.prisma/client`에 Prisma Client가 에러 없이 생성되고, TypeScript 타입이 정상 추론된다.

**Scenario 2: datasource provider 전환 검증**
- Given: `schema.prisma`의 provider가 `sqlite`로 설정된 상태
- When: provider를 `postgresql`로 변경하고 `DATABASE_URL`을 Supabase 연결 문자열로 교체한 뒤 `npx prisma generate`를 실행하면
- Then: Prisma Client가 에러 없이 재생성되고, PostgreSQL 전용 타입(JSONB 등)이 반영된다.

**Scenario 3: 환경변수 분리 검증**
- Given: `.env`(로컬)와 `.env.production`(운영)이 각각 존재하는 상태
- When: 각 환경에서 `DATABASE_URL`을 로드하면
- Then: 로컬은 `file:./dev.db`, 운영은 `postgresql://...` 형식이 올바르게 로드된다.

## ⚙️ Technical & Non-Functional Constraints
- **CON-07**: Next.js App Router + Prisma ORM 스택 고정 — 다른 ORM(TypeORM, Drizzle 등) 사용 금지
- **CON-08**: JSONB 필드 4개(`answers`, `report`, `ai_output`, `hallucination_tags`)의 로컬↔운영 호환성 유의
- **ASM-05**: Prisma ORM이 SQLite↔PostgreSQL 간 스키마 호환성을 보장한다는 가정 하에 진행
